# Summarize Work Skill

## Description

总结当前会话的工作内容，并将总结写入项目根目录下的 markdown 文档。

## When to Use

当用户想要保存或记录当前会话的工作成果时，使用此 skill。

## Usage

```
skill: summarize-work
```

## Workflow

1. 分析当前会话的对话历史，提取关键工作内容：
   - 完成了哪些任务
   - 修改了哪些文件
   - 解决了什么问题
   - 实现了什么功能

2. 生成结构化的工作总结，包含：
   - 工作标题/主题
   - 完成时间
   - 主要内容摘要
   - 修改的文件列表
   - 关键决策和解决方案

3. 将总结内容写入项目根目录下的 `work-summary-{timestamp}.md` 文件

## Output Format

生成的 markdown 文件包含以下结构：

```markdown
# 工作总结: {主题}

**时间**: {YYYY-MM-DD HH:MM}

## 概述

{工作内容摘要}

## 完成的任务

- [x] 任务1
- [x] 任务2

## 修改的文件

| 文件 | 操作 | 说明 |
|------|------|------|
| path/to/file | 创建/修改/删除 | 说明 |

## 关键决策

{重要的技术决策和原因}

## 后续工作

{建议的后续任务或注意事项}
```

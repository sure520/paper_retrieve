# 论文总结奖励函数实现

我已经成功实现了用于强化学习训练的论文总结奖励函数，以及相关的模型训练代码适配。以下是完成的主要工作：

## 1. 创建奖励函数文件

创建了 `train/reward.py` 文件，实现了奖励函数的核心逻辑，包括：

- **辅助函数**：
  - `nli_entailment_score`：判断假设是否被前提蕴含
  - `call_qwen_max`：调用Qwen-Max API进行打分
  - `extract_top_tfidf_keywords`：提取关键词
  - `translate_zh_to_en`：中英文翻译
  - `extract_proposed_method_name`：提取论文提出的新方法名称

- **核心奖励函数**：
  - `compute_reward`：计算综合奖励，包含5个维度的评分
  - `parse_generated_output`：解析生成的JSON输出
  - `paper_summary_reward_function`：论文总结任务的奖励函数接口

## 4. 异常处理机制

为了提高系统的健壮性，添加了异常处理机制：

- 当无法连接到HuggingFace下载NLI模型时，返回默认值
- 当翻译服务不可用时，返回原文本
- 对无效输入进行安全检查，确保奖励函数返回值在[0,1]范围内

## 5. 奖励函数设计

奖励函数根据输出的5个字段，从5个维度进行评分：

| 字段 | 评分维度 | 实现方式 |
|------|----------|----------|
| `summary` | 准确性和完整性 | NLI模型判断 + 规则模板检查 |
| `algorithm` | 清晰度和完整性 | 关键词覆盖 + LLM-as-a-Judge |
| `compare_result` | 对比方法和效果差异 | 模式匹配 + 数字引用检查 |
| `keyword_problem` | 问题类关键词准确性 | TF-IDF词重合度 + 领域通用术语检查 |
| `keyword_algorithm` | 算法类关键词准确性 | 新方法名包含检查 |

总奖励 = w₁R₁ + w₂R₂ + ... + w₅R₅，初始权重相等。

## 6. 技术选型

- **NLI模型**：使用 `MoritzLaurer/DeBERTa-v3-base-mnli-fever-anli` 模型
- **关键词提取**：使用 `sklearn` 的 `TfidfVectorizer`
- **翻译**：使用 `googletrans` 库
- **LLM调用**：使用阿里云DashScope API调用Qwen-Max

这个奖励函数可以用于GRPO等强化学习算法，训练模型提升结构化论文总结能力。通过对同一篇论文生成多个不同摘要，然后用自动奖励函数打分，构成group-wise preference，从而训练模型生成更优质的结构化论文总结。
